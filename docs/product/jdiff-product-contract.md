# jdiff Product Contract

Input specification: `/Users/lyo/Downloads/SPEC.md` (Draft SPEC v0.1).

## Product Goal

`jdiff` is a cross-platform desktop tool for inspecting, comparing, and
merging JAR/ZIP archives and folders. Decompiled Java source is read-only.
Merge always copies original entry bytes, never regenerated bytecode.

## Accepted MVP Contract

1. Open `.jar` files, `.zip` files, and folders from a validated path.
2. List entries lazily and classify them as directory, class, text, or binary.
3. Compare two sources by normalized entry path, uncompressed size, and CRC32
   metadata. ZIP/JAR files use archive metadata; folders compute file CRC32
   while indexing.
4. Read text and binary entries. Class decompilation is exposed through a
   sidecar boundary and may degrade independently when the JVM sidecar is not
   installed.
5. Stage copies in either direction and apply all pending copies for a target
   archive or folder.
6. Save archives through a same-directory temporary archive and atomic
   replacement. Save folders by writing staged files through temp files and
   atomic replacement at each relative path. Optional backup creates one
   overwritten `.bak` for archive targets or snapshots the target folder to
   `<folder>.bak`.
7. Warn callers when a staged save changes a signed archive.
8. Search entry paths, text files, and Java class constant-pool UTF-8 values
   without decompilation.

## Planned Desktop Surface

The desktop shell is Tauri v2 with a React + shadcn/ui + Tailwind + Monaco
frontend. The frontend is a view and intent emitter. Rust owns archive state,
staged changes, and bytes.

## Explicit Non-Goals

- Recompiling or editing decompiled Java.
- APK/DEX support.
- Signature stripping.
- Raw compressed-byte copy optimization.
- Native-image sidecar packaging in the MVP.

## Delivered Enhancements Beyond MVP

These post-MVP capabilities are implemented in the current build. They extend
the accepted contract above without changing its non-goals.

- Nested-archive expansion: archives nested inside a source (jar/zip/war/ear)
  expand lazily in the tree, diff through `compute_nested_diff` using the
  `parent!/inner` path separator, and merge by flattening staged replacements
  back into their parent archives.
- Multi-tab diff workspace: up to 10 concurrent diff tabs with LRU eviction and
  per-tab view-mode and preview state, alongside the Files panel.
- Hierarchical file tree with per-node status rollup.
- Config drawer that groups decompiler engine, whitespace, backup, and
  search-scope options away from the operational surface.
- Startup splash screen shown while the JVM sidecar warms.
