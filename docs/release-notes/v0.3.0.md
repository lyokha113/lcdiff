## LCDiff v0.3.0

Release build focused on packaging compatibility for macOS, Arch Linux, and
Ubuntu LTS Linux.

### Downloads

| Platform | File |
| --- | --- |
| macOS (Apple Silicon) | `LCDiff-0.3.0-aarch64.dmg` |
| Linux x86_64 (Ubuntu 24.04 LTS AppImage) | `LCDiff_0.3.0_ubuntu24.04_amd64.AppImage` |
| Linux x86_64 (Ubuntu 24.04 LTS deb) | `LCDiff_0.3.0_ubuntu24.04_amd64.deb` |
| Linux x86_64 (Ubuntu 26.04 LTS AppImage) | `LCDiff_0.3.0_ubuntu26.04_amd64.AppImage` |
| Linux x86_64 (Ubuntu 26.04 LTS deb) | `LCDiff_0.3.0_ubuntu26.04_amd64.deb` |
| Arch Linux | `yay -S lcdiff` from AUR |

### Install

Download the installer helper for your platform alongside the binary, then:

**macOS** (app is unsigned; the script clears the Gatekeeper quarantine flag):

```bash
bash install-macos.sh
```

**Ubuntu Linux**:

```bash
bash install-linux.sh LCDiff_0.3.0_ubuntu24.04_amd64.AppImage
bash install-linux.sh LCDiff_0.3.0_ubuntu24.04_amd64.deb
```

Use the artifact built on the same Ubuntu LTS floor you run, especially for
WebKit/GTK-linked desktop dependencies.

**Arch Linux**:

```bash
yay -S lcdiff
# or: paru -S lcdiff
```

### Notes

- macOS build is **arm64 only** and unsigned (ad-hoc). Intel Macs and signed
  builds should build from source or use the platform validation runbook.
- Linux release builds target **x86_64** and are built separately on Ubuntu
  24.04 LTS and Ubuntu 26.04 LTS to avoid fragile GTK/WebKit dependency drift.
- Arch Linux uses the AUR package rather than a GitHub release binary.
- Linux bundles are unsigned.
