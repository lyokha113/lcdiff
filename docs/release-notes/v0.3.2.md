## LCDiff v0.3.2

Patch release focused on decompiler coverage.

### Changes

- Added JD-Core and JD-Core v0 as selectable Java decompiler engines.
- Kept Vineflower as the default decompiler and CFR as an alternate engine.
- Extended the JVM sidecar smoke test to verify JD-Core and JD-Core v0 runtime
  decompilation.
- Updated sidecar protocol, cache versioning, Preferences persistence, and
  product docs for the expanded engine list.

### Notes

- The sidecar now compiles against Java 17 because the current decompiler
  dependency set requires it; LCDiff continues to bundle a Java 17 runtime for
  release builds.
- Arch Linux users can install or upgrade from AUR:

```bash
yay -S lcdiff
```

- Windows 10/11 builds use the Windows release workflow and remain unsigned
  unless signing secrets are configured.
